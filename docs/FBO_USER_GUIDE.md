# FBO: инструкция для пользователя

## Что делает модуль

Модуль FBO отвечает на три простых вопроса:
1. Какой спрос ожидается по каждому SKU на 30, 60 и 90 дней.
2. В какой кластер нужно везти товар.
3. Сколько товара нужно дослать на FBO уже сейчас.

Результат автоматически:
- показывается в Telegram;
- записывается в Google Sheets.

Важно:
- расчёт выполняется автоматически;
- реальные действия по поставке защищены;
- система не создаёт поставку молча и без подтверждения.

## Как читать цифры

`Demand 30 / 60 / 90`
Сколько штук система ожидает продать за следующие 30, 60 и 90 дней.

`Recommended 30 / 60 / 90`
Сколько штук нужно довезти, чтобы закрыть нужный горизонт.

`Current Stock`
Сколько товара система видит сейчас.

`Stock Days`
На сколько дней хватит текущего остатка при текущем темпе продаж.

`Confidence`
Насколько расчёт надёжен.
Чем выше значение, тем лучше исторические данные.

`Data Quality Note`
Короткое пояснение, как именно считался результат.
Сейчас в базе есть продажи по SKU и дням, но нет прямых продаж по кластерам, поэтому кластерный спрос оценивается через распределение остатков по складам FBO.

## Как пользоваться в Telegram

Откройте Telegram и отправьте команду:

```text
/supply fbo
```

Что вы увидите:
- SKU;
- кластер;
- спрос на 30 / 60 / 90 дней;
- рекомендуемую поставку на 30 / 60 / 90 дней;
- уровень уверенности расчёта.

Дополнительные команды:

```text
/supply warehouses
/supply clusters
/supply orders
```

## Как пользоваться в Google Sheets

Модуль пишет данные в лист:

```text
FBO Demand
```

Главные колонки:
- SKU
- Product
- Cluster
- Warehouse
- Avg Daily Sales
- Current Stock
- Stock Days
- Demand 30 / 60 / 90
- Recommended 30 / 60 / 90
- Confidence
- Data Quality Note

## Как запустить вручную

Показать расчёт на экране:

```bash
python -m ozon_agent.cli supply fbo
```

Показать один SKU:

```bash
python -m ozon_agent.cli supply fbo --sku 123456789
```

Показать больше строк:

```bash
python -m ozon_agent.cli supply fbo --max-rows 100
```

Сразу пересчитать и записать в Google Sheets:

```bash
python -m ozon_agent.cli supply fbo --sync-sheets
```

## Как использовать каждый день

Рекомендуемый порядок работы:
1. Открыть Telegram и выполнить `/supply fbo`.
2. Открыть лист `FBO Demand` в Google Sheets.
3. Отсортировать строки по `Recommended 30` или `Stock Days`.
4. Посмотреть сначала самые критичные SKU.
5. Если поставка нужна, перейти к согласованию и созданию draft.
6. Бронировать слот только после подтверждения владельца.

## Что система делает сама, а что нет

Система делает сама:
- считает спрос;
- считает рекомендуемую поставку;
- записывает расчёт в Google Sheets;
- показывает результат в Telegram.

Система не делает сама без подтверждения:
- создание draft;
- выбор слота;
- создание реальной поставки.

Это значит, что модуль умеет думать автоматически, но не делает логистическое действие без контроля.

## Текущее ограничение

Сейчас модуль не видит прямые продажи по кластерам, потому что таких данных нет в текущей схеме БД.
Поэтому кластерный спрос сейчас оценочный, а не прямой.

Для управленческой работы этого достаточно, чтобы правильно расставлять приоритеты.
Для полной автоматизации следующий шаг — загрузка прямой региональной или кластерной истории спроса, если она будет доступна в потоке данных Ozon.

## Следующий шаг: 1С

Самый безопасный путь интеграции с 1С такой:
1. Выгружать строки FBO в стабильную таблицу или JSON.
2. Передавать в 1С SKU, кластер, рекомендуемое количество и статус.
3. Получать обратно из 1С подтверждённое количество и номер документа.
4. Оставить бронирование слота внутри этого агента, пока интеграция 1С не будет проверена на практике.

Такой порядок безопасен: сначала связывается бизнес-планирование, потом автоматизируются реальные логистические действия.

## Как подтверждать FBO поставки

Порядок действий:
1. Открыть Telegram и выполнить `/supply fbo`.
2. Если рекомендации подходят, выполнить `/supply fbo-propose`.
3. Посмотреть созданные ID через `/supply proposals`.
4. Подтвердить нужный ID: `/supply approve <proposal_id>`.
5. После подтверждения создать draft.
6. Посмотреть слоты и выбрать нужный слот.

Если совсем просто:
- `/supply fbo` = посмотреть, что везти;
- `/supply fbo-propose` = превратить расчёт в предложение;
- `/supply proposals` = увидеть ID предложений;
- `/supply approve <id>` = подтвердить предложение.
